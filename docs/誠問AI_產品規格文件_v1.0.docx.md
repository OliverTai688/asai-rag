# **一、執行摘要  Executive Summary**

誠問 AI 是一套專為保險通訊處與業務員打造的 AI 銷售智能平台，融合 SPIN 國際銷售法、生成式 AI、語音互動、模擬演練、CRM 管理與全頁面 AI 助手，打造保險業史上最完整的業務輔助生態系。

| A精準銷售 以 SPIN 四步驟 AI 引導，讓每次拜訪有方法、有結構、有成果 | D劇場演練 AI 模擬客戶角色，進行文字與語音雙模式對話排練 |
| :---- | :---- |
| **B智能 CRM** 360° 客戶管理，從家庭結構到保障缺口一手掌握 | **E全頁 AI 助手** RAG 知識問答 \+ 客戶洞察 \+ 自然語言網頁操作 |
| **C團隊管理** 通訊處後台，即時掌握業務員績效與客戶互動數據 | **F安全可靠** 雲端加密架構，資料安全符合金融法規要求 |

# **二、產品願景與市場定位**

## **2.1 產品定位**

誠問 AI 不是通用 AI 工具的套殼，而是深度理解保險銷售流程的垂直行業 SaaS 平台。我們的核心信念：

* 每一位業務員都值得擁有 AI 等級的銷售教練

* 每一次客戶接觸都應該有數據驅動的準備與追蹤

* 通訊處主管需要的是可視化的團隊戰力，而非感覺

## **2.2 目標市場**

| 主要市場 | 台灣壽險、產險業務通訊處（預估 3,500+ 處） |
| :---- | :---- |
| **次要市場** | 獨立業務員、保經代公司、區域業務主管 |
| **擴展市場** | 東南亞保險業務市場（2027+ 規劃） |
| **市場規模** | 台灣保險業務員逾 35 萬人；通訊處年均軟體預算 NT$2M+ |

## **2.3 競爭差異化**

市場上現有工具多為通用 AI 問答或單點功能，誠問 AI 的差異化在於：

| A保險垂直深耕 內建台灣保險業術語、法規知識、商品邏輯，非通用 LLM | C全流程閉環 從訪前規劃→SPIN 引導→演練→報告→追蹤，一站完成 |
| :---- | :---- |
| **B語音互動** 業界首創語音模擬演練，貼近真實拜訪情境 | **D行為數據** 客戶閱讀追蹤讓業務員知道客戶「真正在乎什麼」 |

# **四、功能模組規格  Module Specifications**

| 模組 01  SPIN AI 對話引導 *Structured Sales Intelligence | Core Engine* |  |
| :---- | :---- |
| **觸發入口** | 首頁「開始對話」按鈕；客戶檔案→SPIN 引導；訪前規劃延伸 |
| **雙模式** | （A）釐清自我：業務員自省思路整理 ／（B）設計提問：產生客戶提問策略 |
| **SPIN 流程** | Situation（情境）→ Problem（問題）→ Implication（暗示）→ Need-Payoff（需求回報） |

| **輸出連結** | 一鍵進入「AI 報告生成」或「劇場演練」模組 |

| 模組 02  劇場模擬演練 *AI Role-Play Theater | Voice & Text* |  |

| **客戶角色庫** | 根據客戶檔案自動生成客製化 AI 客戶人物；支援手動選擇或自定義角色 |

| **劇本整合** | 自動套用訪前規劃產出的對話劇本；支援自由模式演練 |

# **六、資料治理與隱私合規**

誠問 AI 處理業務員與客戶的敏感個人資料，需嚴格遵循台灣個人資料保護法及金融業相關規範。

| 合規項目 | 設計策略 |
| ----- | ----- |
| **資料最小化** | 僅收集業務必要資料；客戶敏感欄位（身份證字號等）本地端加密後儲存 |
| **存取控制** | RBAC 角色權限；主管只能看自屬業務員數據；跨通訊處資料隔離 |
| **AI 訓練政策** | 客戶資料不用於 LLM 模型訓練；對話記錄匿名化後方可用於產品改善 |
| **資料保存期限** | 對話記錄預設保存 3 年；到期自動刪除或通知客戶 |
| **右利保障** | 支援客戶資料查閱申請、更正申請、刪除申請（Right to Erasure） |
| **稽核日誌** | 所有資料存取操作完整記錄；支援匯出供法遵稽核 |
