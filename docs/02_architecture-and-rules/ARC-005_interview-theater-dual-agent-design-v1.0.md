## 0. 設計總綱

把現有「SPIN 問答」與「劇場演練」重構為**兩個獨立 agent，各自有「訪談蒐集 → 產出」兩階段**，並建立在一個**共用的半結構訪談引擎**之上。

```
                ┌─────────────────────────────────────┐
                │   半結構訪談引擎 (Interview Engine)    │
                │   讀「訪綱配置」→ AI 主導逐段訪談       │
                │   → 即時記錄素材 → 收斂成「產出」       │
                └─────────────────────────────────────┘
                   ▲ 訪綱配置                ▲ 訪綱配置
        ┌──────────┴─────────┐   ┌──────────┴──────────┐
        │ 訪綱A：顧問陪談      │   │ 訪綱B：劇場場域建構    │
        │ → 訪談 Agent        │   │ → 劇場 Agent          │
        │ 產出：輪廓表+準備卡  │   │ 產出：劇場環境建構表    │
        └────────────────────┘   └──────────┬──────────┘
                                            ▼
                              多角色 RPG 劇場（自由對話＋指定對象）
                                            ▼
                              五個質化 AI 視角回饋（不打分）

        ┌───────────────────────────────────────────────┐
        │  兩種運行模式（任一訪綱皆可套）— 即「兩個模板」     │
        │  ① 獨立模式：不綁客戶，對話從零建構                │
        │  ② 客戶資料模式：載入網頁客戶 → 補問缺口 →        │
        │     萃取事實「業務員確認後寫回客戶檔」              │
        └───────────────────────────────────────────────┘
```

### 已確認的產品決策

1. **「兩個 agent 模板」＝兩種運行模式**（獨立模式 / 客戶資料模式），兩份訪綱皆可套用同一引擎。
2. **客戶資料模式**：訪談中萃取的事實（家庭、保單、痛點等）**經業務員確認後寫回客戶檔**（CRM）。
3. **劇場多角色操作**：**自由對話＋可指定對象**；NPC 之間會互相搭話/插話。
4. **角色與人格**：由訪綱推論生成；常見原型由本文件 §3 提供結構化「種子」。
5. **評分**：不打分數，改以 §4 的「五個質化 AI 視角」各自給觀察與建議。
6. **異議與銷售紅線**：採 §5 的研究結果。
7. **RAG**：保留檔案上傳位置，見 §7。

## 1. 半結構訪談引擎（共用核心）

引擎只認識抽象結構：段落 / 核心題(◆) / 追問(↳) / 每段目標來源用途 / 產出 schema。換 agent＝換訪綱配置，不改引擎。

### 1.1 訪綱配置資料結構（草案）

```ts
interface InterviewOutline {
  id: string;
  name: string;                 // 「顧問陪談」/「劇場場域建構」
  role: string;                 // 引導者對誰訪談：「顧問訪業務員」
  framework: "SPIN_HIDDEN" | "SCHEIN_3LAYER";
  principles: string[];         // 各訪綱的「進行原則」/「引導重點」
  segments: InterviewSegment[];
  outputSchema: OutputField[];  // 這份訪綱最後收斂出的產出欄位
}

interface InterviewSegment {
  order: number;
  title: string;                // 「第1段｜先把客戶現況拼出來」
  subtitle?: string;            // 「（顧問框架：Situation 情境）」
  goal: string;                 // 本段目標
  dataSource: string;           // 系統／會議／訪談
  purpose: string;              // 建構用途
  coreQuestions: { id: string; text: string }[];   // ◆ 必問
  followUps:     { id: string; text: string }[];    // ↳ 選用
  guideNote?: string;           // 引導者引導重點框
}

interface OutputField {
  key: string;
  label: string;
  type: "text" | "list" | "table" | "rolecard" | "relationmap" | "spinQuestions" | "pq";
  sourceSegments: number[];     // 由哪幾段素材收斂
}
```

> 兩份訪綱的 ◆/↳ 題目、引導重點、產出欄位皆可直接從來源文件萃取成上述配置（已完成 9 成，待 §6 定稿少數空缺）。

### 1.2 引擎行為

- AI 依 segment 順序主導訪談；◆ 必問、↳ 視回答彈性追問（順序可調）。
- 每段即時把回答結構化進 `outputSchema` 對應欄位。
- 全程白話（訪綱 A 原則：不對業務員提「SPIN」一詞）。
- 訪綱 B 特有：每筆素材標註「已知事實 vs 合理推論」（呼應其「推論的標註原則」）。

## 2. 兩種運行模式（＝兩個模板）

|                | ① 獨立模式               | ② 客戶資料模式                                                                                             |
| -------------- | ------------------------- | ----------------------------------------------------------------------------------------------------------- |
| 起點           | 空白；第0段先問「要談誰」 | 預載 `Client` / `FamilyMember` / `Policy`                                                             |
| 已知 vs 待確認 | 全部現場問                | CRM 已有者自動標「已知」，只追問缺口                                                                        |
| 對話累積       | 存 session                | 累積對話 + 萃取事實，**經確認後**回寫 `Client.aiTags`、家庭成員、保單、痛點                         |
| 寫回機制       | 不寫回                    | 萃取結果先以 UI 卡片列出 → 業務員逐項確認/編輯 → 寫回（沿用 SPIN 既有 aiTags 動態更新模式，加一道確認閘） |
| 適用           | 新名單、臨時練習          | 既有客戶、長期經營                                                                                          |

> 寫回務必區分「事實 vs 推論」：推論項預設不自動寫回，需業務員主動勾選。

## 3. 劇場角色原型庫（種子）

**用途**：角色由訪綱 B 的素材推論生成；當素材不足時，引擎以下列原型作為「種子」補完。每個原型以 Schein 三層次分層，確保角色「有沒說出口的理由」。

> 設計原理：
>
> - **三層必須表面衝突、根部一致**：嘴上說的（信奉價值）與真正驅動的（基本假設）應矛盾，玩家任務是從人造物/口頭禪反推基本假設。
> - **隱藏勝利條件 = 觸及基本假設**：在價值層硬碰（「其實沒那麼貴」）通常失敗；真正成交來自化解底層恐懼。
> - **異議常是煙霧彈**：「要問家人」「再想想」多為轉移責任/信任未建立的訊號。

### 原型速查（8 種）

| 原型             | DISC   | 核心恐懼（基本假設層） | 典型異議                         |
| ---------------- | ------ | ---------------------- | -------------------------------- |
| 扛家計的支柱型   | D(+C)  | 我倒下家就垮           | 我再想想／預算有其他優先         |
| 精算質疑型       | C      | 被坑／做錯決定         | 太複雜／已有保單且自認夠         |
| 情感顧家型       | I+S    | 顧不好家人＝失職       | 我要跟先生討論                   |
| 謹慎拖延型       | S→C   | 選錯被套牢             | 我考慮一下／最近忙               |
| 忙碌沒空型       | D      | 不想面對自身脆弱       | 最近真的太忙／之後再說           |
| 被排除的配偶型   | S      | 另一半不在時我一無所知 | 我跟先生/太太討論後決定緩緩      |
| 怕拖累子女長輩型 | S→C   | 成為累贅＝失敗         | 不要為我花錢（含忌諱包裝）       |
| 唱反調親戚型     | D 或 C | 失去影響力／傷人情     | 我有朋友也在賣／我親戚理賠很麻煩 |

### 角色卡欄位（每個原型/生成角色皆用此結構）

- 原型名稱
- 在家庭/場域中的常見身分
- 公開立場（信奉的價值，會說的話）
- 沒說出口的（基本假設／深層恐懼）
- 外顯行為（人造物層，行為與口頭禪）
- 情緒觸發點（防禦 / 軟化 / 激動）
- 對保險的核心態度與典型異議
- DISC 傾向
- 多角色劇場中與其他角色的典型張力

### 原型細節（節錄三例，完整八例見研究附錄）

**原型 1｜扛家計的支柱型**

- 身分：家中主要收入者、要保人、重大財務決策者。
- 公開立場：「我是負責任的人，家裡靠我，不用你們操心。」
- 沒說出口的：「萬一我倒下這個家會垮——而我不敢去想那天。」把「提供者」等同自我價值。
- 外顯行為：談數字與責任、少談情緒、獨自赴約、口頭禪「這我來決定就好」。
- 觸發點：防禦＝直白談「你身故後」；軟化＝「家人在你不在時仍能照常生活」；激動＝暗示他「沒準備＝不負責任」。
- DISC：D(+C)。張力：與「被排除的配偶型」——獨斷決策埋下日後翻案伏筆。

**原型 3｜情感顧家型**

- 身分：配偶/母親、家庭日常財務長、需求發起者。
- 公開立場：「只要對孩子、對家裡好，錢不是問題。」
- 沒說出口的：「我得把每個人顧好，否則就是失職」；對家庭破碎有深層焦慮。
- 外顯行為：主動分享家庭故事、易被故事打動、口頭禪「為了孩子」。
- 觸發點：軟化＝真實家庭故事；落淚＝孩子可能失依；防禦＝被當「不懂理財」。
- DISC：I+S。張力：與「精算質疑型」配偶＝憑感覺 vs 要數據的經典夫妻拉鋸。

**原型 8｜唱反調的親戚／職場鏡子型**

- 身分：親戚、同事、「家裡已有做保險的朋友」；典型 gatekeeper/blocker。
- 公開立場：「我是為你好、怕你被騙。」
- 沒說出口的：「這單成了顯得我沒用／失去影響力／傷了我跟業務朋友的人情。」
- 外顯行為：引用理賠被拒故事、質疑動機、搶話替當事人發言。
- 觸發點：防禦＝被排除；激動＝影響力被無視；軟化＝被請教、被給面子。
- DISC：D 或 C。張力：全場最強「會後否決者」，須先處理面子與動機而非辯產品。

> 完整 8 原型九欄位、DISC×恐懼對照、來源與證據分級，見本文件末 §附錄A。

## 4. 五個質化 AI 視角（取代評分）

演練結束後，**不打分數**。改由五個不同背景的 AI 視角各自生成一段質化觀察與建議。五者角度差異明顯、互補；可直接做成五段 prompt。

| 視角         | 看的東西                                 | 站在誰立場           | 會踩煞車       |
| ------------ | ---------------------------------------- | -------------------- | -------------- |
| 教練的耳朵   | 對話結構、探詢深度（SPIN 層次推進）      | 銷售方法論           | 否             |
| 客戶的眼睛   | 此刻情緒、信任/戒備溫度                  | 客戶內心（第一人稱） | 否             |
| 沉默裡的需求 | 不在場家人、未說出口的需求與關係張力     | 整個家庭系統         | 否（看更深）   |
| 守門的良心   | 誠實、揭露、尊重決策節奏、合規           | 公平待客             | **會**   |
| 決策的橋     | 決策摩擦、選項結構、下一步、把大決定拆小 | 客戶的決策能力       | 部分（防操弄） |

**互補性示範**：同一句「這個方案只有今天有」——

- 教練的耳朵：與探詢無關，略過
- 客戶的眼睛：她僵住、戒備上升
- 沉默裡的需求：沒處理到她背後的家人
- 守門的良心：製造假急迫、違反尊重節奏（紅線）
- 決策的橋：用壓力取代降低摩擦、退路被關

### 各視角 prompt 規格

**① 教練的耳朵（The Coach's Ear）**

- 立場：資深銷售教練，只看「對話結構與探詢深度」。
- 觀察重點：是否在客戶說出問題前就推商品；有無問「影響題(Implication)」；客戶是否用自己的話說出痛點與價值；有無先養大問題代價再給方案；說/聽比例。
- 語氣範例：「你在客戶提到『每月對帳忙不完』時碰到一個真實問題，但緊接著跳去介紹商品。試著多留一層：先問『這樣下來有沒有讓你錯過什麼更重要的事？』把代價問出來再帶方案。」
- 理論：Rackham/Huthwaite SPIN selling。

**② 客戶的眼睛（The Customer's Eyes）**

- 立場：完全以「客戶當下內心感受」第一人稱敘事，不評論技巧。
- 觀察重點：客戶放鬆/防備、答案變長或變短；業務有無反映情緒；有無評價語言（你應該…）；有無製造急迫致反彈；客戶問題有無被真正回答。
- 語氣範例：「當你慢下來問她真正擔心什麼，她明顯放鬆了，回答變長、變私人，好像那刻才覺得把顧慮說出口是安全的。」
- 理論：NVC（觀察 vs 評價）、Rogers 積極傾聽、Brehm 心理抗拒；情緒真誠度比正向度更重要。

**③ 沉默裡的需求（The Unspoken in the Room）— 產品核心**

- 立場：受家庭系統與敘事治療訓練者，專聽「沒被說出口的話」與不在場的家人。
- 觀察重點：有無探詢「還有誰被影響」；客戶帶情緒重量提到家人時有無轉向；有無幫客戶說出未說出口的需求；有無察覺關係張力；有無用觀點提問把不在場家人請進對話。
- 語氣範例：「她提到母親三次、有一次停頓很久，你每次都把話帶回保費。那停頓裡可能藏著真正的需求——讓她坐在你面前的，是母親，不是那張保單。」
- 理論：Bowen 家庭系統、White & Epston 敘事治療（缺席卻隱含/雙重傾聽）、Milan 循環提問。

**④ 守門的良心（The Conscience at the Gate）**

- 立場：盡責公正的合規/倫理觀察者，只問「這對待公不公平、合不合規」。
- 觀察重點：是否誇大/暗示保證；是否略過除外費用解約金風險；是否混淆商品本質；是否製造假急迫/利用恐懼；有無確認客戶聽懂；推薦是否符合說出口的需求（適合度）；客戶要「想想/問家人」有無被尊重。
- 語氣範例：「客戶問報酬時你說成保證的。更透明也對她更安全的做法，是清楚區分『保證』與『預估非保證』，別讓她抱著可能落空的期待做決定。」
- 理論：台灣金管會公平待客原則（適合度/告知揭露/廣告真實）；對照 NAIC 最佳利益、FCA Consumer Duty、IDD；猶豫期機制。
- **與 §5 紅線連動**：此視角是偵測 §5「銷售紅線」的主要載體。

**⑤ 決策的橋（The Bridge to Decision）**

- 立場：懂行為經濟學者，看「有無合乎倫理地降低決策摩擦」，嚴守協助 vs 操弄界線。
- 觀察重點：選項是否過載（尤其客戶偏好不確定/選項難比較時）；有無圍繞說出口的需求組織選項；有無清楚「建議下一步」；有無把大決定拆小步驟；客戶猶豫時是加資訊還是找真正卡點；框架是否誠實、退路是否保留。
- 語氣範例：「你一口氣攤開五個方案，她隨即安靜——那常是過載不是興趣。收斂到符合她需求的兩個，會讓選擇變得『做得到』。」
- 理論：Iyengar & Lepper 選擇過載（含 Chernev & Böckenholt 條件性修正）、展望理論、Thaler & Sunstein 推力、nudge vs sludge 倫理界線。

### prompt 撰寫警示（研究中的可信度提醒）

- 「選項越少越好」非鐵律，僅在「偏好不確定/選項難比較」時成立——③⑤ 用語要寫成情境性而非定律。
- 傾聽/說話比例、非語言線索有文化差異——用「可能/似乎」推測語氣，符合「質化不評分」初衷。

## 5. 異議庫 × 銷售紅線

### 5.1 常見異議庫（12 種）

> 核心：多數第一線拒絕不是真異議，而是情緒緩衝或認知失調下的自我合理化。處理原則＝先辨真假、先建風險觀念、再給選擇，而非逐句反駁。人格對應沿用既有 Theater personas。

| #  | 異議              | 客戶說法                   | 背後擔憂                         | 回應方向                                       | 常見人格     |
| -- | ----------------- | -------------------------- | -------------------------------- | ---------------------------------------------- | ------------ |
| 1  | 價格太貴          | 「保費太貴負擔不起」       | 現金流被佔焦慮、未認同價值       | 不爭辯貴/便宜，換算＋對齊預算＋拆解選項        | CONSERVATIVE |
| 2  | 已經有保了        | 「我已經有保險了」         | 不想被重複推銷、不確定保什麼     | 肯定既有，改以保單健檢/缺口檢視切入            | SKEPTICAL    |
