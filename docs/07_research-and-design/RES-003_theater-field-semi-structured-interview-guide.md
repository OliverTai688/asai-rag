**模擬劇場・場域建構半結構訪綱**

以 Schein 組織文化三層次，建構客戶場域與人際連結，供劇場演練使用

故事共同建構　×　多重角色視角　×　系統／會議／訪談三來源

# **一、這份訪綱的目的**

這份半結構訪綱用來為「模擬劇場演練」蒐集並建構素材。它的任務不是查事實，而是把一位客戶身處的「場域」立體地重建出來，讓劇場裡的角色能真實地被扮演、被理解。

* 在人際關係的維度上，發現角色之間的連結，包含表面看不見的隱藏關係。

* 透過故事共同建構與不同角色的視角，看見人的感受、情境與需求。

* 讓保險業務員更理解自己的工作場域——也就是客戶身處的那張關係網。

* 用 Schein 三層次把「表象」一路挖到「深層」，讓後續的劇場推論更貼近現實。

# **二、方法與理論基礎**

### **故事共同建構（劇場視角）**

訪談不是單向蒐集資料，而是由引導者與業務員一起，把客戶的世界「說成一個故事」；過程中刻意切換不同角色（客戶、配偶、同事、主管……）的視角來看同一個情境，讓被忽略的感受、需求與隱藏關係浮現。

### **Schein 組織文化三層次（挖深的工具）**

Schein 把文化分成三層，由淺到深。我們借用它，把客戶的場域也分三層來問，避免只停在表面。

| 層次 | 定義 | 在客戶場域中看什麼 | 例（陳志明場域） |
| :---- | :---- | :---- | :---- |
| **人造物** **Artifacts** | 看得見、聽得到的表層：行為、空間、物件、慣例。 | 客戶的日常、互動對象、可觀察的行為與物件。 | 加班、家庭分工、保單放著沒看。 |
| **信奉的價值** **Espoused Values** | 公開宣稱的目標、規則與「應該」。 | 客戶與環境「說」自己重視什麼。 | 「家庭最重要」「男人要扛起來」。 |
| **基本假設** **Basic Assumptions** | 沒人明說、視為理所當然的深層信念。 | 真正驅動行為、卻不被質疑的前提。 | 「示弱＝失職」「談保險＝觸霉頭」。 |

| 為什麼一定要挖到基本假設　劇場要逼真，角色的行為必須有「沒說出口的理由」。只停在人造物層，角色會像紙片人，演起來假；挖到基本假設，角色才會自己「動起來」，後續推論也才站得住。 |
| :---- |

### **三種資料來源（三角檢證）**

| 資料來源 | 能取得什麼 | 主要對應層次 |
| :---- | :---- | :---- |
| **系統資料（CRM／保單／基本資料）** | 事實、歷史、家庭與受益人結構等硬資料。 | 人造物 |
| **會議資料（晨會／個案討論／陪訪紀錄）** | 同仁怎麼談這位客戶、團隊強調什麼、過往互動。 | 人造物＋價值 |
| **半結構訪談（本訪綱）** | 感受、關係、未明說的假設——只有對話才挖得出來。 | 價值＋基本假設 |

# **三、半結構訪談本體**

以下分七段。◆ 核心題每次必問，↳ 追問依現場彈性使用，順序可調。每段標明「目標／資料來源／建構用途」，右欄記錄蒐集到的素材。

  **第 0 段｜場域聚焦**  

*（暖身）*

**目標：**選定要建構的客戶與場域範圍，確認手上有哪些資料。

**資料來源：**系統＋會議　**建構用途：**界定劇場的範圍與焦點

| 類型 | 引導者提問 | 建構素材記錄 |
| :---- | :---- | :---- |
| **◆** **核心題** **（必問）** | **我們這次要模擬的是哪一位客戶？要建構他的哪些場域？（家庭？職場？還是兩者交會處？）** |  |
| **◆** **核心題** **（必問）** | **為什麼選這個場域？你最想在劇場裡看清楚什麼？** |  |
| **↳** **追問** **（選用）** | 這個場域裡，最關鍵的兩三個人是誰？ |  |
| **↳** **追問** **（選用）** | 你手上有哪些資料可用？（系統、會議紀錄、你自己的觀察） |  |

  **第 1 段｜角色與關係盤點**  

*（人際關係維度）*

**目標：**列出場域中的角色，畫出彼此的連結與張力，特別標出隱藏關係。

**資料來源：**系統（家庭／受益人結構）＋訪談　**建構用途：**角色清單與關係連結圖

| 類型 | 引導者提問 | 建構素材記錄 |
| :---- | :---- | :---- |
| **◆** **核心題** **（必問）** | **這個場域裡有哪些人？各自跟客戶是什麼關係？** |  |
| **◆** **核心題** **（必問）** | **這些人之間，誰跟誰比較親近？誰跟誰之間有張力或距離？** |  |
| **◆** **核心題** **（必問）** | **有沒有「表面上沒事、其實藏著什麼」的關係？** |  |
| **↳** **追問** **（選用）** | 客戶最在意誰的看法？最不想讓誰失望？ |  |
| **↳** **追問** **（選用）** | 哪一段關係，是客戶自己可能都還沒說清楚的？ |  |

  **第 2 段｜人造物層**  

*（Artifacts・看得見的表層）*

**目標：**蒐集可觀察的日常、行為、物件、慣例，作為劇場的布景與動作。

**資料來源：**系統＋會議＋觀察　**建構用途：**場景布景、角色的外顯行為

| 類型 | 引導者提問 | 建構素材記錄 |
| :---- | :---- | :---- |
| **◆** **核心題** **（必問）** | **客戶的一天或一週，大概是什麼樣子？** |  |
| **◆** **核心題** **（必問）** | **在這個場域裡，有哪些反覆出現的行為或慣例？ 範例：如客戶會一直查詢，故意問他知道的問題、如護士常進行研究臨床的經驗。** |  |
| **◆** **核心題** **（必問）** | **有哪些具體的物件、空間或文件，是這個場域的象徵？（如沒看的保單、加班的辦公室、牆上的全家福）** |  |
| **↳** **追問** **（選用）** | 從系統或會議資料看，有哪些「事實」可以直接搬上劇場？ |  |
| **↳** **追問** **（選用）** | 客戶說話、做事的方式有什麼特徵？（口頭禪、習慣動作） |  |

  **第 3 段｜信奉的價值層**  

*（Espoused Values・公開說的「應該」）*

**目標：**找出客戶與環境公開宣稱重視的東西、規則與信念，並注意言行落差。

**資料來源：**訪談＋會議　**建構用途：**角色的台詞與立場、衝突的伏筆

| 類型 | 引導者提問 | 建構素材記錄 |
| :---- | :---- | :---- |
| **◆** **核心題** **（必問）** | **在這個場域裡，大家或客戶會說「我覺得...」「應該…」「就是...」的是什麼？** |  |
| **◆** **核心題** **（必問）** | **客戶會用什麼話來形容自己的角色或責任？** |  |
| **◆** **核心題** **（必問）** | **這個場域有沒有不成文、但大家都認同的規則？** |  |
| **↳** **追問** **（選用）** | 這些「說出來的價值」，跟他實際的行為一致嗎？哪裡有落差？ |  |
| **↳** **追問** **（選用）** | 落差的地方，往往就是劇場張力的來源——你觀察到哪些？ |  |

  **第 4 段｜基本假設層**  

*（Basic Assumptions・沒人明說的深層）*

**目標：**挖出驅動行為、卻被視為理所當然、沒人質疑的深層前提。

**資料來源：**訪談（主要靠對話推敲）　**建構用途：**角色行為的內在引擎，讓模擬逼真

| 類型 | 引導者提問 | 建構素材記錄 |
| :---- | :---- | :---- |
| **◆** **核心題** **（必問）** | **在這個場域裡，有哪些事是「不用說、大家都默認」的？** |  |
| **◆** **核心題** **（必問）** | **客戶有哪些行為，表面理由之外，可能有更深、沒說出口的原因？** |  |
| **◆** **核心題** **（必問）** | **有什麼話題或情緒，是這個場域裡「碰不得」的？** |  |
| **↳** **追問** **（選用）** | 如果有人挑戰這個默認，會發生什麼？大家會怎麼反應？ |  |
| **↳** **追問** **（選用）** | 這個假設是怎麼形成的？（過去的經驗、家庭、職場文化） |  |

| 推論的標註原則　基本假設多半是「推論」出來、而非客戶直接說出的。提問時用「我猜會不會是…」的試探語氣；記錄時務必區分哪些是『已知事實』、哪些是『合理推論』，避免把假設當定論帶進劇場。 |
| :---- |

  **第 5 段｜故事共同建構**  

*（多重角色視角）*

**目標：**選一個具體事件，從不同角色的視角重述，浮現感受、需求與隱藏連結。

**資料來源：**訪談　**建構用途：**劇場的核心場景與要探索的問題

| 類型 | 引導者提問 | 建構素材記錄 |
| :---- | :---- | :---- |
| **◆** **核心題** **（必問）** | **選一個最近發生、最能代表這個場域的具體事件。當時發生了什麼？** |  |
| **◆** **核心題** **（必問）** | **如果由「客戶本人」來說這件事，他會怎麼說？他的感受是什麼？** |  |
| **◆** **核心題** **（必問）** | **換成「另一個關鍵角色」（如配偶或同事）來說，同一件事會變成什麼樣子？** |  |
| **↳** **追問** **（選用）** | 兩個版本有什麼不同？差異透露了什麼隱藏的關係或需求？ |  |
| **↳** **追問** **（選用）** | 這個場景裡，誰的需求沒有被說出來、也沒有被滿足？ |  |

  **第 6 段｜轉化為劇場設定**  

*（收斂）*

**目標：**把以上素材收斂成一個可立刻演練的場景設定。

**資料來源：**綜合　**建構用途：**劇場腳本與角色簡報

| 類型 | 引導者提問 | 建構素材記錄 |
| :---- | :---- | :---- |
| **◆** **核心題** **（必問）** | **我們要演的是哪一個場景？地點、時間、在場的有哪些人？** |  |
| **◆** **核心題** **（必問）** | **這場戲要探索的核心問題或張力是什麼？** |  |
| **◆** **核心題** **（必問）** | **每個角色帶著什麼立場、什麼沒說出口的東西進場？** |  |
| **↳** **追問** **（選用）** | 業務員會扮演哪個角色？想透過扮演看懂什麼？ |  |
| **↳** **追問** **（選用）** | 演完後，我們想得到什麼領悟，帶回真實的拜訪？ |  |

# **四、產出：劇場環境建構表（範例：陳志明）**

七段訪談後，把素材整理成下列四份產出，交給劇場演練使用。以下為範例客戶「陳志明」的填寫示意。

### **4-1　場域概述**

建構家庭場域與職場場域的交會處。焦點人物：陳志明，38 歲科技業中階主管、家庭主要經濟支柱；近期同事重病、母親住院，正評估換房。

### **4-2　角色卡**

| 角色（與客戶關係） | 公開立場（價值） | 沒說出口的（基本假設／感受） | 外顯行為（人造物） |
| :---- | :---- | :---- | :---- |
| **陳志明（焦點）** | 「我要扛起這個家。」 | 怕示弱、怕一談「萬一」就成真。 | 加班、把保單放著不看。 |
| **小婷（配偶）** | 「家裡的事我們一起。」 | 覺得被排除在財務決策外，擔心卻不說。 | 兼職、私下查資料。 |
| **母親（長輩）** | 「不要為我花錢。」 | 怕成為兒子的負擔。 | 生病了卻說「沒事」。 |
| **生病的同事（職場鏡子）** | 「我也以為我還年輕。」 | 對陳志明像一面鏡子，逼他面對脆弱。 | 突然請假、離開職場。 |
| **主管／職場文化** | 「拚才有未來。」 | 默認「不能示弱、不能請假」。 | 績效導向、加班常態。 |

### **4-3　關係連結與隱藏張力**

| 關係配對 | 連結與隱藏張力 |
| :---- | :---- |
| **陳志明 ↔ 小婷** | 表面分工清楚，實則財務由陳一人扛；隱藏張力：太太想參與卻沒被邀請。 |
| **陳志明 ↔ 母親** | 互相體貼卻互相隱瞞（都說「沒事」）；隱藏連結：都怕成為對方的負擔。 |
| **陳志明 ↔ 同事** | 同事生病是觸發點；鏡子關係，逼陳面對自己也可能倒下的脆弱。 |
| **陳志明 ↔ 職場** | 職場「不能示弱」的基本假設外溢到家庭，使他不敢正視保障缺口。 |

### **4-4　三層次摘要**

| 層次 | 摘要 |
| :---- | :---- |
| **人造物** | 加班、保單沒看、母親住院、同事離開職場、看房計畫。 |
| **信奉的價值** | 「家庭最重要」「我要扛起來」「不要造成別人負擔」。 |
| **基本假設** | 「示弱＝失職」「談風險＝觸霉頭」「財務是我一個人的事」。 |

### **4-5　待演練的核心場景與張力問題**

* 場景：陳志明下班回家，太太提起母親的醫藥費，兩人之間出現一段短暫的沉默。

* 核心問題：當「我要扛」遇上「其實快扛不動、又不敢說」，這個家要如何開口對話？

* 業務員扮演：旁觀者或太太，練習在沉默裡看見「沒說出口的需求」。

| 這套訪綱最終想達成的　當業務員能在劇場裡，從不同角色的眼睛看見同一個家，他回到真實拜訪時問的就不再是「你要不要保險」，而是「我看見你一直在扛，要不要我陪你一起看看，怎麼扛得久一點」。 |
